Hi, I’m **Gustra**. I build software that is meant to be used, not just demoed: village platforms, BUMDes operations, billing systems, accounting backends, geospatial interfaces, internal dashboards, and product infrastructure that can survive real deployment conditions.

Most of my recent work sits at the intersection of **civic tech**, **operational SaaS**, **accounting workflows**, and **field-ready web systems**.

## What I work on

- **Village digital ecosystems** with modular services, shared auth, centralized data, and map-based information layers.
- **BUMDes and billing platforms** for recurring charges, tenant flows, service administration, and operational reporting.
- **Accounting systems** spanning custom UI, backend APIs, database design, and ERP integration.
- **Geospatial and public information tools** for village assets, land reference views, public facilities, tourism, and field data collection.
- **Practical product infrastructure** covering Docker, PostgreSQL, Redis, tunnels, deployment structure, and server-first thinking.

## Selected projects from my local workspace

### 1. Desa Gulingan Digitalisasi
An integrated village platform with an API gateway and multiple services for user management, survey, absensi, guest book, CCTV, SOS, and map central.

**Stack:** Node.js, Fastify, Prisma, PostgreSQL, Redis, OpenAPI, PWA  
**Repo:** [gustradev/desa-gulingan-digitaliasasi](https://github.com/gustradev/desa-gulingan-digitaliasasi)

### 2. TPS3R Billing / SIERA
Multi-tenant billing platform for village and TPS3R services covering invoices, customer registry, payment flows, reporting, and audit-oriented architecture.

**Stack:** NestJS, TypeScript, Prisma, PostgreSQL, Redis, Docker  
**Repo:** [gustradev/tps3rBilling](https://github.com/gustradev/tps3rBilling)

### 3. Accounting System Backend & Database
Private accounting backend monorepo with API, worker, shared packages, schema management, migrations, and seed flows.

**Stack:** NestJS, Prisma, PostgreSQL, Docker Compose, TypeScript  
**Repo:** [gustradev/accounting-system-backend-db](https://github.com/gustradev/accounting-system-backend-db)

### 4. Project Akunting

**Stack:** React, Mantine, Vite PWA, NestJS, ERPNext

### 5. Accounting Sederhana / BUMDes

## How I usually build

```text
Understand the real workflow
-> model the data clearly
-> design the service boundaries
-> make the deployment path realistic
-> keep the UI understandable for operators
-> ship something that can be maintained
```

I care about:

- clear system boundaries
- field-usable UX
- backend correctness
- private/self-hosted deployment options
- scaling architecture only where it is justified
